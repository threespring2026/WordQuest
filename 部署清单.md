# WordQuest 部署清单（小白版）

> 本文档帮助你把本地的 WordQuest 项目部署到线上，让任何人都能通过网址访问。
> 整个过程需要注册 2 个网站：GitHub（代码托管）和 Vercel（网站部署）。
> 预计首次部署耗时：30-60 分钟。之后每次更新只需 2-3 分钟。

---

## 一、准备工作

### 你需要准备的东西

| 项目 | 说明 |
|------|------|
| 电子邮箱 | 用于注册账号，推荐 Gmail 或 QQ 邮箱 |
| WordQuest 项目文件夹 | 就是你现在的 `/Volumes/SC/work/AI/program/WordQuest` |
| 网络 | 需要能访问 GitHub 和 Vercel（如果打不开，可能需要科学上网） |

---

## 二、第一步：注册 GitHub 账号

GitHub 是一个代码托管平台，你的代码会存放在这里。

### 2.1 打开 GitHub 官网

1. 打开浏览器
2. 访问：**https://github.com**
3. 如果页面是英文，不用担心，按下面步骤操作即可

### 2.2 注册账号

1. 点击页面右上角的 **Sign up**（注册）按钮
2. 填写信息：
   - **Enter your email**：输入你的邮箱地址
   - **Create a password**：设置一个密码（至少 8 位，建议包含数字和字母）
   - **Enter a username**：设置用户名（英文，比如 `zhangsan123`，这个会出现在你的网址里）
3. 点击 **Continue**（继续）
4. 完成人机验证（可能让你选图片或拼图）
5. 点击 **Create account**（创建账号）
6. 去邮箱查收验证邮件，点击邮件里的验证链接

### 2.3 登录确认

1. 回到 https://github.com
2. 点击 **Sign in**（登录）
3. 输入邮箱和密码
4. 看到右上角出现你的头像，说明登录成功 ✅

---

## 三、第二步：下载 GitHub Desktop（推荐小白使用）

GitHub Desktop 是一个图形界面工具，让你不用敲命令就能上传代码。

### 3.1 下载安装

1. 打开：**https://desktop.github.com**
2. 点击 **Download for macOS**（下载 Mac 版本）
3. 下载完成后，打开 `.dmg` 文件
4. 把 **GitHub Desktop** 图标拖到 **Applications**（应用程序）文件夹
5. 打开「应用程序」文件夹，双击 **GitHub Desktop** 启动

### 3.2 登录 GitHub Desktop

1. 启动后会看到欢迎页面
2. 点击 **Sign in to GitHub.com**
3. 浏览器会自动打开，点击 **Authorize desktop**（授权）
4. 回到 GitHub Desktop，看到你的用户名说明登录成功 ✅

### 3.3 配置 Git 信息

1. 在 GitHub Desktop 菜单栏点击 **GitHub Desktop** → **Preferences**（偏好设置）
2. 点击 **Git** 标签
3. 填写：
   - **Name**：你的名字（中英文都可以）
   - **Email**：你注册 GitHub 的邮箱
4. 点击 **Save**（保存）

---

## 四、第三步：把项目上传到 GitHub

### 4.1 添加本地项目

1. 在 GitHub Desktop 菜单栏点击 **File** → **Add Local Repository**
2. 点击 **Choose...**（选择）
3. 找到你的项目文件夹：`/Volumes/SC/work/AI/program/WordQuest`
4. 点击 **Add Repository**

**如果提示「This directory does not appear to be a Git repository」：**
- 点击 **create a repository**（创建仓库）
- Name（名称）填：`WordQuest`
- 其他保持默认
- 点击 **Create Repository**

### 4.2 首次提交代码

1. 在左下角的 **Summary** 框里输入：`首次提交`
2. 点击蓝色的 **Commit to main** 按钮
3. 等待提交完成（可能需要几秒到几分钟，取决于文件大小）

### 4.3 推送到 GitHub

1. 点击顶部的 **Publish repository**（发布仓库）
2. 在弹出的窗口里：
   - **Name**：保持 `WordQuest`
   - **Keep this code private**：
     - ☑️ 勾选 = 私有仓库（只有你能看）
     - ☐ 不勾选 = 公开仓库（任何人都能看代码）
   - 推荐：**不勾选**（公开），这样 Vercel 免费版可以正常使用
3. 点击 **Publish Repository**
4. 等待上传完成（首次可能需要 1-3 分钟）

### 4.4 确认上传成功

1. 打开浏览器，访问：`https://github.com/你的用户名/WordQuest`
2. 如果能看到你的代码文件列表，说明上传成功 ✅

---

## 五、第四步：注册 Vercel 账号

Vercel 是一个网站托管平台，可以把你的代码自动变成一个可访问的网站。

### 5.1 打开 Vercel 官网

1. 打开浏览器
2. 访问：**https://vercel.com**

### 5.2 使用 GitHub 登录（推荐）

1. 点击页面右上角的 **Sign Up**（注册）
2. 选择 **Continue with GitHub**（使用 GitHub 登录）
3. 如果弹出授权页面，点击 **Authorize Vercel**（授权）
4. 看到 Vercel 仪表盘页面，说明登录成功 ✅

---

## 六、第五步：在 Vercel 部署项目

### 6.1 导入 GitHub 项目

1. 在 Vercel 仪表盘，点击 **Add New...** → **Project**
2. 在 **Import Git Repository** 部分，找到你的 `WordQuest` 项目
3. 点击旁边的 **Import** 按钮

**如果看不到你的项目：**
- 点击 **Adjust GitHub App Permissions**
- 在 GitHub 页面勾选你的仓库
- 点击 **Save**（保存）
- 回到 Vercel 刷新页面

### 6.2 配置项目（保持默认即可）

1. **Project Name**：保持 `wordquest` 或自定义（这会影响你的网址）
2. **Framework Preset**：选择 **Other**（其他）
3. **Root Directory**：保持 `./`（不用改）
4. **Build Command**：留空（不用填）
5. **Output Directory**：留空（不用填）

### 6.3 部署

1. 点击 **Deploy** 按钮
2. 等待部署完成（通常需要 30 秒到 2 分钟）
3. 看到 **Congratulations!** 页面说明部署成功 ✅

### 6.4 获取你的网址

部署成功后，Vercel 会给你一个网址，格式类似：
- `https://wordquest-xxx.vercel.app`

点击这个网址，如果能看到你的 WordQuest 游戏，说明部署完成！🎉

## 七、日常更新代码（每次修改后）

以后每次修改代码，只需要做以下 3 步：

### 7.1 步骤一：保存修改

在你的代码编辑器（Cursor）里保存所有修改的文件（快捷键 `Cmd + S`）

### 7.2 步骤二：提交并推送

1. 打开 **GitHub Desktop**
2. 左侧会显示你修改了哪些文件（Changed Files）
3. 在左下角的 **Summary** 框里简单描述你改了什么，比如：
   - `修复登录问题`
   - `更新词库`
   - `调整界面样式`
4. 点击 **Commit to main** 按钮
5. 点击顶部的 **Push origin** 按钮

### 7.3 步骤三：等待自动部署

1. 推送完成后，Vercel 会自动检测到更新
2. 自动开始重新部署（通常 30 秒 - 2 分钟）
3. 部署完成后，刷新你的网站就能看到更新了

**查看部署状态：**
- 打开 https://vercel.com/dashboard
- 点击你的项目
- 可以看到部署进度和状态

## 八、常见问题

### Q1：GitHub 或 Vercel 打不开怎么办？

**可能原因**：网络限制
**解决方法**：
- 尝试使用 VPN 或科学上网工具
- 或者使用手机热点试试

### Q2：部署后网站打不开或显示错误？

**排查步骤**：
1. 在 Vercel 仪表盘查看部署日志，看是否有报错
2. 确保 `index.html` 文件在项目根目录
3. 检查是否有文件路径错误（大小写敏感）

### Q3：修改代码后网站没更新？

**可能原因**：浏览器缓存
**解决方法**：
- 强制刷新：`Cmd + Shift + R`
- 或清除浏览器缓存

### Q4：想要自定义域名怎么办？

1. 在 Vercel 项目设置里点击 **Domains**
2. 添加你购买的域名
3. 按提示配置 DNS 解析

### Q5：忘记 GitHub 密码怎么办？

1. 访问 https://github.com/password_reset
2. 输入你的邮箱
3. 按邮件提示重置密码

## 十一、安全提醒

⚠️ **不要把以下内容上传到公开仓库：**
- API 密钥（如 `sk-xxx` 开头的字符串）
- 数据库密码
- 个人敏感信息

你的 `ai.config.js` 中包含 SiliconFlow 的 API Key，如果仓库是公开的，建议：
1. 把 API Key 移到 Vercel 的环境变量中
2. 或者把仓库设为私有
