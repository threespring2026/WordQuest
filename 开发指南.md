## 1. 项目概述

| 字段 | 说明 |
|------|------|
| 项目类型 | 基于 Web 的英语词汇学习互动小游戏 |
| 前端技术栈 | HTML5 + 原生 JavaScript + Tailwind CSS（单文件实现，无需构建工具） |
| 后端技术栈 | Vercel Serverless Functions（无需租用服务器）+ Supabase（数据库与用户认证 BaaS） |
| 核心目标 | 通过剧情对话、互动游戏与多维反馈，帮助玩家在真实语境中记忆和理解 3～10 个英语单词 |
| DEMO 范围 | 模块三至模块五完整可运行；模块一、模块二仅完成接口预留，界面不开发 |
| 部署成本 | 初期完全免费（Vercel 免费套餐 + Supabase 免费套餐均可满足 DEMO 与小规模运营需求） |

## 2. 技术架构总览

### 2.1 核心技术选型

本项目采用 **"无服务器（Serverless）"** 架构，不需要租赁或自行维护服务器。所有基础设施均由第三方平台托管，开发者只需专注于编写业务代码。

| 层级 | 选用方案 | 作用 | 费用 |
|------|---------|------|------|
| 前端托管 | **Vercel** | 托管 HTML5 游戏，代码推送 GitHub 后自动发布 | 免费套餐可用 |
| 后端逻辑 | **Vercel Serverless Functions** | AI 接口代理、配额校验等服务端逻辑（几十行代码，无需配置服务器） | 含于 Vercel 免费套餐 |
| 数据库 | **Supabase**（PostgreSQL） | 存储用户、单词本、故事配置文档等所有数据 | 免费套餐含 500 MB 数据库 |
| 用户认证 | **Supabase Auth**（内置） | 注册、登录、JWT Token 全部由 Supabase 托管处理，无需自己开发 | 含于 Supabase 免费套餐 |
| 后台管理 | **Supabase Studio**（内置）+ 轻量管理页 | 数据库可视化管理界面，免开发即可用；Prompt 管理通过前端内嵌管理页实现 | 免费 |
| AI 接口 | **DeepSeek**（推荐）/ OpenAI / 其他兼容 OpenAI 格式的服务商 | 故事生成能力，密钥存储于 Vercel 环境变量 | 按调用量付费（DeepSeek 费用约为 OpenAI 的 1/10） |

### 2.2 架构示意图

```
┌──────────────────────────────────────────────────────────┐
│                    前端（Web 客户端）                        │
│   HTML5 + 原生 JS + Tailwind CSS                          │
│   托管于 Vercel，GitHub 推送后自动构建并发布                  │
└────────────┬─────────────────────────┬────────────────────┘
             │ HTTPS（游戏数据读写）       │ HTTPS（AI 生成请求）
             ▼                          ▼
┌────────────────────────┐  ┌───────────────────────────────┐
│  Supabase（BaaS 平台）  │  │  Vercel Serverless Functions  │
│  ┌─────────────────┐   │  │                               │
│  │ PostgreSQL 数据库 │   │  │  /api/story/synopsis          │
│  │ 用户 / 单词本     │   │  │  /api/story/config            │
│  │ 故事配置文档      │   │  │  /api/admin/prompts           │
│  │ Prompt 模板      │   │  │                               │
│  └─────────────────┘   │  │  AI API 密钥存储于              │
│  ┌─────────────────┐   │  │  Vercel 环境变量（安全隔离）      │
│  │ Supabase Auth   │   │  └───────────────┬───────────────┘
│  │ 用户认证 / JWT   │   │                  │ HTTPS
│  └─────────────────┘   │                  ▼
│  ┌─────────────────┐   │  ┌───────────────────────────────┐
│  │ Supabase Studio │   │  │      第三方 AI API             │
│  │ 内置数据管理界面  │   │  │   DeepSeek（推荐）/ OpenAI    │
│  └─────────────────┘   │  └───────────────────────────────┘
└────────────────────────┘
```

### 2.3 设计原则

- **密钥安全：** AI API 密钥存储于 Vercel 平台的环境变量中，仅在 Serverless Function 运行时读取，绝不出现在前端代码或 API 响应中。
- **数据归属：** 每一份生成的故事配置文档在存储时自动绑定至发起请求的 Supabase Auth 用户 ID。
- **接口版本化：** 所有自定义 API 路径以 `/api/v1/` 开头，便于后续迭代。
- **认证外包：** 用户注册、登录、Token 签发与验证全部由 Supabase Auth 处理，无需自行编写认证逻辑。
- **DEMO 降级：** 当 AI 接口调用连续失败 3 次时，Serverless Function 返回预置的 DEMO 故事配置，保障演示可正常进行。

## 3. 视觉与 UI 设计规范

- **整体风格：** 2D 现代、圆润、可爱像素风格。
- **3D 按钮：** 带底部边框（`box-shadow: 0 4px 0 [darkerColor]`），点击时有物理下沉按压反馈（`:active` 状态 `transform: translateY(4px)` + 阴影消失）。
- **色彩搭配：** 明亮柔和（护眼绿、天空蓝、明黄色），背景以白色和浅灰色为主。
- **移动端优先：** PC 端居中显示为模拟手机屏幕的圆角矩形容器，完美适配移动端浏览器。

## 5. 模块一：用户系统与登录

> **DEMO 版本：本模块界面不开发。** 服务端接口和数据结构完整设计并预留，前端 DEMO 阶段以游客身份绕过，直接进入模块三。

### 5.1 用户类型与权限

| 权限项 | 普通用户 | VIP 用户 |
|--------|---------|---------|
| 注册赠送体验次数 | 3 次 | 无限制 |
| 次数耗尽后的补充规则 | 每自然周（周一 00:00 重置）补充 1 次 | 无限制 |
| 单词本服务端存储 | 不支持（本地临时） | 支持（服务端持久化） |
| 故事配置文档存储 | 不支持 | 自动存储，可查看历史记录 |
| 角色自定义 | 仅默认随机角色 | 解锁全部角色风格与部件 |

> **次数计算规则：** 初始 3 次与每周补充次数共用同一计数器，消耗完毕后需等待下一自然周周一 00:00 重置。次数重置时间以服务端时间为准。

### 5.2 角色系统

- 系统在注册时随机分配初始角色，每个角色拥有独特的专属颜色标识。
- VIP 用户支持角色风格设定与部件自定义，部件包括：发型、服装、配件等独立模块，可自由组合。

### 5.3 其他功能

- **英语水平：** 注册时选择（初级 / 中级 / 高级），用于后续故事生成时控制对话的语言难度。
- **打卡日历：** 展示用户的游戏打卡记录。
- **充值页面：** 支持用户升级为 VIP。
- **内置词库：** 服务端维护核心词库，供游戏准备模块快速选词。

## 6. 模块二：游戏准备

> **DEMO 版本：本模块界面不开发。** 使用服务端预置的 DEMO 固定单词包代替，前端直接跳过本模块进入模块三。

### 6.1 单词包建立

- 玩家手动输入 3～10 个英语单词。
- 输入后前端调用服务端词典查询接口，自动展示每个单词的释义与音标。
- 所有确认的单词存入用户的"单词本"，按创建时间自动分组（VIP 用户服务端持久化，普通用户本地临时存储）。

### 6.2 单词本分组管理

- 以输入时间为维度自动分组，组名默认为创建日期（如 `2026-02-26`）。
- 支持查看历史单词包、重新发起故事生成。

### 6.3 复习机制

- 依据**艾宾浩斯遗忘曲线**（采用 SM-2 算法），服务端在适当时间推送复习提醒。
- 复习题型：单词闪卡、选义匹配等轻量互动题型（详细设计在正式版迭代中实现）。

## 7. 模块三：故事生成

> **DEMO 版本固定单词包：**
> `reputation / royal / relieve / plenty / govern / frustrate / enable / division / disorder / destruction`

### 7.1 流程概述

```
用户进入模块三
  │
  ▼
前端发起请求 → 服务端调用 AI 接口生成"故事简介"
  │
  ▼
向玩家展示故事简介，等待确认
  │
  ├─ 玩家点击"重新生成" → 重新调用 AI 接口（不限次数）
  └─ 玩家点击"确认故事" → 服务端调用 AI 接口生成"故事配置文档"
                            │
                            ▼
                       服务端校验配置文档合法性
                            │
                            ├─ 校验通过 → 存入数据库（绑定至当前用户账号）→ 返回配置 ID 给前端
                            └─ 校验失败 → 重试（最多 3 次）→ 失败则返回预置 DEMO 配置
```

### 7.2 故事简介字段定义

| 字段名 | 类型 | 说明 | 长度限制 |
|--------|------|------|---------|
| `background` | string | 故事发生的环境与时代、主角经历的事情及当前处境（中文） | ≤ 40 字 |
| `mission` | string | 主角需完成的一个明确、短小的任务，如送信、寻找物品等（中文） | ≤ 20 字 |

### 7.3 故事配置文档完整字段定义

AI 根据已确认的故事简介，生成以下完整的配置文档。此文档经服务端校验后存入数据库，并作为模块四的唯一数据来源。

**顶层字段：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `mapId` | integer | 地图编号，取值范围 1～6，对应地图资源表 |
| `mood` | string | 故事情绪，枚举值：`funny`（搞笑）/ `warm`（温馨）/ `sad`（悲伤）/ `happy`（快乐） |
| `npcs` | array | 出场 NPC 列表，固定 4 个元素，见下方 NPC 字段定义 |
| `dialogues` | array | 对话列表，固定 6 组，见下方对话字段定义 |
| `wordPack` | array | 本局使用的单词包，`WordItem[]` 结构，透传至模块四供高亮与释义使用 |

**NPC 元素字段（`npcs[]`）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `npcId` | string | NPC 编号，取值范围 `NPC_01`～`NPC_06` |
| `slot` | integer | 在当前地图中的站位编号，取值范围 1～4，4 个 NPC 的 slot 值不重复 |

**对话元素字段（`dialogues[]`）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `round` | integer | 对话轮次，从 1 连续递增至 6 |
| `npcId` | string | 本轮对话发起的 NPC 编号 |
| `npcLine` | string | NPC 第一句台词（英文），用于引出情境；单词包中的词须用 `**词**` 标注 |
| `npcLineCN` | string | NPC 第一句台词对应的中文翻译 |
| `options` | array | 玩家回答选项，固定 3 个元素，见下方选项字段定义 |
| `correctReply` | string | NPC 正确回应台词（英文），在玩家选对后展示，并引导玩家前往下一个 NPC（台词中须提及下一个 NPC 的名字） |
| `correctReplyCN` | string | NPC 正确回应台词对应的中文翻译 |

**选项元素字段（`options[]`）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | string | 选项标识，固定为 `A`、`B`、`C` |
| `text` | string | 选项文本（英文）；正确选项须包含关键词并语境合理，错误选项须与关键词语义相反或语境无关 |
| `isCorrect` | boolean | 是否为正确答案，3 个选项中有且仅有 1 个为 `true` |
| `errorReply` | string \| null | 选择此选项时 NPC 的错误回应台词（英文）；正确选项此字段为 `null` |

**剧本配置表格示例（完整一行）：**

| 轮次 | NPC编号 | NPC第一句（英） | NPC第一句（中） | 选项A（英） | 选项B（英） | 选项C（英） | 正确选项 | 对A的错误回应（英） | 对B的错误回应（英） | NPC正确回应（英） | NPC正确回应（中） |
|------|---------|--------------|--------------|----------|----------|----------|--------|----------------|----------------|--------------|--------------|
| 1 | NPC_03 | The kingdom needs someone to **govern** with wisdom. | 王国需要一个有智慧的人来**治理**。 | I will rule with fear. | I will **govern** with justice and care. | I don't care about the kingdom. | B | That's not leadership, that's tyranny. | Indifference will only bring **disorder**. | Well said. The people will trust you. Go speak with Emily at the harbor. | 说得好，人民会信任你的。去海港找 Emily 谈谈吧。 |

### 7.4 配置文档服务端校验规则

| 校验项 | 规则 |
|--------|------|
| `mapId` | 必须是 1～6 的整数 |
| `mood` | 必须是 `funny`、`warm`、`sad`、`happy` 其中之一 |
| `npcs` | 必须恰好 4 个；`npcId` 在 `NPC_01`～`NPC_06` 范围内；`slot` 值不重复且在 1～4 内 |
| `dialogues` | 必须恰好 6 组；`round` 字段从 1 连续递增；每组 NPC 均出现在 `npcs` 列表中 |
| `options` | 每组对话的 `options` 恰好 3 个；`id` 为 `A`、`B`、`C`；`isCorrect` 有且仅有 1 个为 `true` |
| 关键词标注 | 所有 `npcLine`、`options[].text`、`correctReply` 字段中，单词包内的词必须被 `**` 包裹 |
| NPC 覆盖 | 所有 4 个出场 NPC 均至少参与 1 轮对话 |

## 8. 模块四：沉浸式剧情互动

### 8.1 游戏界面初始化

1. 前端通过配置 ID 从服务端获取完整的故事配置文档。
2. 按 `mapId` 加载对应地图背景图。
3. 玩家角色出现在画面**下方固定起始点**，角色头顶显示玩家昵称（DEMO 版本显示"玩家"）。
4. 4 个 NPC 按 `slot` 编号出现在地图对应位置，头顶显示角色头衔与名字。
5. 第 1 轮对话对应的 NPC 头顶显示 ❗ 感叹号图标，引导玩家前往。

### 8.2 移动与交互规则

| 操作 | 行为 |
|------|------|
| 点击地图空白区域 | 玩家角色以直线移动至目标位置（DEMO 版本，无碰撞检测） |
| 点击带 ❗ 的 NPC | 玩家角色自动走向该 NPC，到达后自动触发对话 |
| 点击无 ❗ 的 NPC | 无响应（当前轮次未轮到该 NPC） |

> **顺序强制线性：** 玩家必须按感叹号指引依次与 NPC 交互，不可跳过或乱序触发。

### 8.3 对话交互机制

**气泡布局：**

- NPC 的对话气泡：显示在该 NPC 头顶上方。
- 玩家的选项气泡：显示在玩家角色正下方。

**每组对话完整流程：**

```
① NPC 说第一句（npcLine，英文）
   └─ 气泡中显示"翻译"按钮，点击可展开 npcLineCN
② 展示玩家三个选项（options[].text）
    ✅ 选择正确 → 显示 NPC 正确回应（correctReply）
                  → 感叹号移至下一 NPC（若为第 6 轮则触发结算）
    ❌ 选择错误 → NPC 展示对应错误回应（option.errorReply）+ 惊讶表情
                  → 去除该选项 → 玩家继续从剩余选项中作答（错误次数 +1）
                  → 直至作答正确
③ 进入下一轮
```

### 8.4 单词辅助功能

| 功能 | 说明 |
|------|------|
| 关键单词高亮 | 单词包中的单词在所有对话文本中以**红色加粗**显示 |
| 悬浮释义 | 对话文本中任意单词鼠标悬停（或移动端长按）时，显示该词的词性、中文释义与音标 |
| 翻译按钮 | 每段 NPC 台词下方附"翻译"按钮，点击展开对应中文翻译；关键单词在翻译文本中同样红色加粗 |
| 单词发音 | 点击任意单词时自动朗读该单词（详见第 14 节） |

## 9. 模块五：评分与结算

所有 6 轮对话全部完成后，自动弹出评分与结算面板。

### 9.1 评分规则

| 错误次数 | 星级评分 |
|---------|---------|
| 0 次 | ⭐⭐⭐ 三星（满星） |
| 1～2 次 | ⭐⭐ 两星 |
| 3 次及以上 | ⭐ 一星 |

> 评分结果与错误次数在结算后存储至服务端的游戏记录中，绑定至当前用户账号。

### 9.2 结算面板内容

- 星级评分展示（带逐星亮起动效）。
- 错误次数统计文本（如"本局共犯 2 次错误"）。
- 恭喜完成图标与提示文字。

### 9.3 后续操作选项

| 操作按钮 | 行为说明 |
|---------|---------|
| 📖 复习单词表 | 弹出本局所有单词列表，含词性、中文释义与音标 |
| 🔄 再玩一次 | 前端重新拉取同一故事配置文档，重置所有游戏状态，重新开始 |

## 10. 服务端设计

### 10.1 服务端职责与实现方式

| 职责 | 实现方式 | 说明 |
|------|---------|------|
| 用户认证 | **Supabase Auth（内置，无需开发）** | 注册、登录、JWT Token 签发与验证均由 Supabase 平台处理 |
| 配额管理 | **Vercel Serverless Function** | 调用 AI 前查询 `users` 表中的 `quota_used` 字段，判断是否允许本次请求 |
| AI 代理 | **Vercel Serverless Function** | 从 Vercel 环境变量读取 AI API 密钥，发起 AI 调用，结果返回给前端 |
| 数据存储 | **Supabase PostgreSQL（内置，无需开发）** | 通过 Supabase 客户端 SDK 进行所有增删改查操作 |
| Prompt 管理 | **Supabase 数据库表 + 轻量管理页** | Prompt 模板存储于数据库，通过前端管理页或 Supabase Studio 编辑 |
| AI 密钥管理 | **Vercel 环境变量仪表盘** | 直接在 Vercel 项目设置中更新密钥值，无需重新编写代码，修改后重新部署生效 |

## 11. 后台管理系统

### 11.1 系统概述

后台管理采用**双轨制**方案，最大程度减少自行开发工作量：

| 管理工具 | 负责范围 | 是否需要开发 |
|---------|---------|------------|
| **Supabase Studio**（平台内置，免费） | 用户信息查看与修改、故事配置文档查看、数据库直接操作、角色权限设置 | 无需开发，开箱即用 |
| **前端内嵌管理页**（`/admin` 路径） | Prompt 模板管理（新增、切换启用版本）、用户等级批量调整 | 需少量开发（约 200 行代码） |
| **Vercel 控制台**（平台内置，免费） | AI API 密钥的新增、更新与删除 | 无需开发，直接在网页配置 |

管理员角色通过 `users.role` 字段区分，前端根据此字段控制管理入口的可见性。管理员分为两个级别：

| `role` 值 | 权限说明 |
|-----------|---------|
| `super_admin` | 全部管理功能，包括修改其他管理员的角色 |
| `admin` | 用户管理、故事配置查看、Prompt 管理 |

### 11.2 功能模块

#### 11.2.1 AI API 密钥管理

**管理工具：** Vercel 控制台（无需开发）

**操作路径：** 登录 [vercel.com](https://vercel.com) → 选择项目 → Settings → Environment Variables

**操作规范：**

- 新增 / 修改密钥：在 Vercel 控制台编辑 `AI_API_KEY`、`AI_BASE_URL`、`AI_MODEL` 三个环境变量，保存后触发自动重新部署。
- 删除密钥：在 Vercel 控制台删除对应环境变量，重新部署后失效。
- 切换 AI 服务商：只需修改上述三个变量的值（例如将 DeepSeek 换为 OpenAI），代码不需要做任何修改。

**安全保障（Vercel 平台提供）：**

- 环境变量在 Vercel 平台加密存储，部署后在 UI 界面不可读取（只能覆盖写入）。
- 环境变量仅在 Serverless Function 运行时注入，不出现在前端代码或构建产物中。
- 不写入日志，不出现在任何 API 响应中。

#### 11.2.2 Prompt 模板管理

**功能需求：**

- 分"故事简介生成（synopsis）"和"故事配置生成（config）"两种类型，各自独立管理。
- 展示所有历史版本 Prompt，标注当前启用版本。
- 支持新增 Prompt：填写模板名称、选择类型、编写正文（富文本或 Markdown 编辑器），提交后保存为新版本。
- 支持切换"当前启用模板"：同一类型同时只能有 1 个模板处于启用状态。
- 支持查看历史模板的完整内容（只读）。
- 不支持直接修改已有模板（需新建版本替代），以保证历史配置文档可追溯其生成所用的 Prompt 版本。

**Prompt 占位符规范：**

| 占位符 | 说明 | 适用类型 |
|--------|------|---------|
| `{words}` | 当前单词包的单词列表（逗号分隔） | synopsis、config |
| `{synopsis}` | 已确认的故事简介 JSON（background + mission） | config |
| `{english_level}` | 用户英语水平（beginner / intermediate / advanced） | synopsis、config |
| `{npc_list}` | 当前可用 NPC 列表（编号 + 名称 + 性格） | config |
| `{map_list}` | 当前可用地图列表（编号 + 场景名称） | config |

#### 11.2.3 用户账号管理

**管理工具：** Supabase Studio（无需开发）+ 前端 `/admin` 页（需少量开发）

**Supabase Studio 可直接操作（无需开发）：**

- 在 `users` 表中查看、筛选所有用户信息（昵称、等级、配额、注册时间等）。
- 直接在表格界面修改用户的 `tier`（免费 ↔ VIP）、`role`（权限）、`quota_used`（重置配额归零）字段。
- 在 Supabase Auth 控制台禁用 / 启用用户账号（`Disable user` 按钮），禁用后该账号无法登录。
- 关联查询用户的单词本、故事配置文档、游戏记录（通过 SQL 编辑器执行 JOIN 查询）。

**前端 `/admin` 页需支持（需少量开发）：**

- 用户列表展示与搜索，支持按昵称、邮箱、等级筛选。
- 一键修改用户等级与重置配额（通过 Supabase SDK 写入）。

#### 11.2.4 故事配置文档管理

**管理工具：** Supabase Studio（无需开发）

**Supabase Studio 可直接操作（无需开发）：**

- 在 `story_configs` 表中查看所有生成的配置文档列表（配置 ID、用户 ID、生成时间、单词包概要）。
- 点击任意行查看 `config_json` 字段的完整 JSON 内容（只读，用于内容质量审查）。
- 支持按 `user_id`、`created_at` 字段筛选。
- 不支持编辑或删除（保证数据完整性和可追溯性）。

### 13.2 DEMO 开发阶段 Mock 策略

**核心原则：先把游戏本体开发测试完毕，再接入真实 AI。** 在 AI 未接入前，用固定的预置文本替代 AI 返回值，确保游戏完整流程可以运行和测试。

**实现方式：** 在 Serverless Function 中增加 Mock 开关。

```javascript
// api/v1/story/synopsis.js 示例结构
export default async function handler(req, res) {

  // Mock 开关：环境变量 USE_MOCK_AI=true 时直接返回固定数据
  if (process.env.USE_MOCK_AI === 'true') {
    return res.json(MOCK_SYNOPSIS)  // 返回下方预置数据
  }
```

### 13.5 AI 接口容错策略

| 异常情况 | 处理策略 |
|---------|---------|
| HTTP 4xx（认证失败、密钥无效） | 立即终止重试，记录错误日志，向前端返回明确错误提示，管理员需检查后台密钥配置 |
| HTTP 5xx（服务商内部错误） | 等待 2 秒后重试，最多重试 3 次 |
| 响应非合法 JSON | 尝试提取响应中的 JSON 块；失败则视为无效响应，执行重试 |
| 配置文档校验失败 | 记录失败字段，重新发起生成请求，最多重试 3 次 |
| 3 次重试全部失败 | 服务端返回预置 DEMO 故事配置文档（固定内容），前端显示提示"正在使用演示故事" |

## 14. 音效与发音功能规范

### 14.1 单词发音

- **实现方案：** 优先使用浏览器原生 **Web Speech API**（`speechSynthesis`），英文语言设置为 `en-US`，语速（rate）设为 `0.85`。
- **触发方式：** 点击对话气泡中任意单词的音标展示区域，或点击词典浮层中的发音图标。
- **降级方案：** 若当前浏览器不支持 Web Speech API，发音图标隐藏，功能不可用（不报错）。

### 14.2 背景音乐

- 游戏场景内持续播放与地图情绪对应的循环背景音乐（BGM）。
- 进入结算界面时，BGM 渐出，播放结算音乐。
- 玩家可通过界面角落的音量图标切换静音 / 播放状态，设置存储至 `localStorage`，下次进入游戏时保持用户偏好。

### 14.3 音效

| 触发场景 | 音效 |
|---------|------|
| 点击按钮 | 轻快点击音（`sfx_click.mp3`） |
| 对话气泡弹出 | 弹出提示音（`sfx_bubble.mp3`） |
| 选择正确选项 | 正向反馈音（`sfx_correct.mp3`） |
| 选择错误选项 | 错误提示音（`sfx_wrong.mp3`） |
| 结算面板出现 | 完成庆祝音（`sfx_complete.mp3`） |
| 星级逐个亮起 | 星星亮起音（`sfx_star.mp3`，每亮一颗触发一次） |

## 15. 非功能性需求

### 15.1 安全性

- **用户认证：** 由 Supabase Auth 全权负责，JWT Token 的签发与验证均在 Supabase 平台内完成。前端 Serverless Function 通过 Supabase Admin SDK 验证请求中携带的 Token，无需自行实现认证逻辑。
- **管理员权限校验：** Vercel Serverless Function 在处理 `/api/v1/admin/` 路径的请求前，须验证 Token 对应用户的 `role` 字段值为 `admin` 或 `super_admin`，否则返回 403。
- **AI API 密钥：** 存储于 Vercel 环境变量，由 Vercel 平台加密托管，绝不出现在代码、日志、数据库或 API 响应中。
- **用户密码：** 由 Supabase Auth 以 bcrypt 加密存储，开发者无需关心。
- **数据库行级安全（RLS）：** 在 Supabase 中为所有业务表开启 Row Level Security，确保用户只能读写属于自己的数据。

### 15.2 数据完整性

- 故事配置文档一经存储不允许修改或删除，以保证历史游戏记录可追溯。
- Prompt 模板不允许原地修改，需新建版本替代，历史模板保留。
- `story_configs` 表的 `prompt_template_id` 字段须记录生成时实际使用的 Prompt 版本。

### 15.3 性能要求

- 前端首屏加载时间（TTI）目标：4G 网络环境下 ≤ 3 秒。
- AI 接口调用超时设置：单次请求超时阈值为 30 秒。
- 数据库 `users`、`story_configs` 表须对 `user_id`、`created_at` 字段建立索引。

### 15.4 兼容性

- 前端最低支持：Chrome 90+、Safari 14+、Firefox 90+。
- 移动端支持：iOS 14+ Safari、Android Chrome 90+。
- 不依赖任何前端框架或构建工具，确保单文件 HTML 可直接在浏览器中打开运行。

### 16.3 详细步骤

#### 步骤一：初始化 Supabase 数据库

1. 登录 [supabase.com](https://supabase.com)，点击 **"New project"**，填写项目名称（如 `wordquest`），选择离用户最近的服务器地区，设置数据库密码（妥善保存）。

2. 等待项目创建完成（约 1 分钟）后，进入 **"SQL Editor"**，依次执行以下建表 SQL：

```sql
-- 用户扩展信息表（关联 Supabase Auth 内置用户表）
create table public.users (
  id uuid references auth.users(id) on delete cascade primary key,
  nickname text,
  tier text default 'free' check (tier in ('free', 'vip')),
  role text default 'user' check (role in ('user', 'admin', 'super_admin')),
  english_level text default 'beginner' check (english_level in ('beginner', 'intermediate', 'advanced')),
  avatar_color text default '#4A90E2',
  avatar_config jsonb,
  quota_used integer default 0,
  quota_reset_at timestamptz default (now() + interval '7 days'),
  created_at timestamptz default now()
);

-- 单词本分组表
create table public.word_books (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users(id) on delete cascade,
  label text,
  created_at timestamptz default now()
);

-- 单词条目表
create table public.word_items (
  id uuid default gen_random_uuid() primary key,
  word_book_id uuid references public.word_books(id) on delete cascade,
  word text not null,
  phonetic text,
  part_of_speech text,
  definition text,
  added_at timestamptz default now()
);

-- Prompt 模板表
create table public.prompt_templates (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  type text not null check (type in ('synopsis', 'config')),
  content text not null,
  is_active boolean default false,
  created_by uuid references public.users(id),
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

-- 故事配置文档表
create table public.story_configs (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users(id),
  word_book_id uuid references public.word_books(id),
  synopsis_background text,
  synopsis_mission text,
  config_json jsonb not null,
  prompt_template_id uuid references public.prompt_templates(id),
  created_at timestamptz default now()
);

-- 游戏记录表
create table public.game_sessions (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.users(id),
  story_config_id uuid references public.story_configs(id),
  error_count integer default 0,
  stars integer check (stars in (1, 2, 3)),
  completed_at timestamptz default now()
);

-- 为高频查询字段建立索引
create index on public.story_configs(user_id, created_at);
create index on public.game_sessions(user_id);
create index on public.word_items(word_book_id);
```

5. 进入 **"Authentication → Policies"**，为所有业务表开启 Row Level Security（RLS）并添加基础策略：用户只能读写自己的数据（`auth.uid() = user_id`）。

### 16.5 后续迭代与维护

| 操作 | 方法 |
|------|------|
| 更新前端页面或游戏逻辑 | 修改代码 → 在 GitHub Desktop 中 Commit + Push → Vercel 自动重新部署（约 1 分钟） |
| 更换 AI API 密钥 | 登录 Vercel 控制台 → Project Settings → Environment Variables → 修改 `AI_API_KEY`（和 `AI_BASE_URL`、`AI_MODEL`）→ 触发重新部署 |
| 修改 Prompt 模板 | 登录游戏管理员账号 → 访问 `/admin` → Prompt 管理页新增版本并切换启用 |
| 调整用户权限 | 登录 Supabase Studio → `users` 表 → 直接修改 `tier` 或 `role` 字段 |
| 查看生成内容 | 登录 Supabase Studio → `story_configs` 表 → 查看 `config_json` 字段 |
| 数据库备份 | Supabase Studio → Database → Backups（免费套餐每日自动备份，保留 7 天） |
