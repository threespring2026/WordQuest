## 一、这个项目是什么

**《词境历险：像素语者》（WordQuest）** —— 一款基于 Web 的英语词汇学习小游戏。

玩家输入一组英语单词，AI 自动生成一个专属的冒险故事，玩家扮演主角在地图上与 NPC 对话、做多项选择题，在真实语境中记住这些单词。

```
你输入单词 → AI 生成故事 → 玩家进入地图 → 与 NPC 对话 → 回答选择题 → 完成游戏 → 评分结算
```

**核心亮点：**
- 每次游戏都是 AI 为你定制的故事，不重复
- 单词在对话中以红色加粗显示，鼠标悬停显示释义
- 完成后有星级评分和单词复习表，形成学习闭环

## 四、游戏的 5 个模块（全局地图）

```
模块一          模块二          模块三          模块四              模块五
[用户登录]  →  [游戏准备]  →  [故事生成]  →  [剧情互动]    →  [评分结算]
 账号/权限       输入单词        AI生成故事       在地图上对话        星级评分
 ⚠️DEMO不做     ⚠️DEMO不做      ✅ DEMO做        ✅ DEMO做           ✅ DEMO做
```

### 模块一：用户登录（DEMO 不开发）

- 普通用户：注册送 3 次游玩，耗尽后每周补充 1 次
- VIP 用户：无限次，可存储单词本和游戏历史
- 角色系统：随机分配角色颜色，VIP 可自定义角色外观
- **DEMO 处理：** 跳过登录，直接以游客身份进入模块三

### 模块二：游戏准备（DEMO 不开发）

- 玩家输入 3～10 个英语单词，查询释义，建立"单词包"
- 单词存入"单词本"，按日期分组管理
- 按遗忘曲线推送复习提醒（SM-2 算法）
- **DEMO 处理：** 跳过，使用以下固定单词包：
  > `reputation / royal / relieve / plenty / govern / frustrate / enable / division / disorder / destruction`

### 模块三：故事生成（DEMO 开发）

**这是 AI 的核心工作。** 两步生成：

1. **生成故事简介** → 展示给玩家确认（背景 ≤40字，任务 ≤20字）
2. **生成故事配置文档** → 包含地图选择、NPC 选择、6 组完整对话剧本

配置文档格式严格，服务端会自动校验，不合法时最多重试 3 次，失败则用预置 DEMO 配置兜底。

### 模块四：沉浸式剧情互动（DEMO 开发）

- 玩家出现在地图下方，4 个 NPC 分布在地图各处
- 带 ❗ 感叹号的 NPC 是当前要交互的，点击自动走过去
- 对话格式：NPC说 → 玩家三选一 → NPC回应，共 6 轮线性推进
- 选错时 NPC 显示惊讶表情，去掉错误选项，继续作答（错误次数计入结算）
- 单词包中的词红色加粗，鼠标悬停显示释义，可点击翻译按钮

### 模块五：评分与结算（DEMO 开发）

- 0 错误 = ⭐⭐⭐，1-2 次 = ⭐⭐，3次以上 = ⭐
- 三个按钮：📖 复习单词表 / 🔄 再玩一次 / 🏠 返回

## 七、建议的开发顺序

按以下顺序开发，可以最快得到一个可以运行的 DEMO：

```
第一步：准备工作（约 2 小时）
  ├─ 注册 GitHub / Vercel / Supabase 账号（暂时不需要 DeepSeek 账号）
  ├─ 在 Supabase 中建好所有数据库表（运行开发指南中的 SQL 脚本）
  └─ 创建 GitHub 仓库，连接 Vercel，配置环境变量（此阶段设 USE_MOCK_AI=true）

第二步：写 AI 代理接口框架 + Mock（约 2 小时）
  ├─ 写 /api/v1/story/synopsis.js（含 Mock 开关，USE_MOCK_AI=true 时返回固定数据）
  └─ 写 /api/v1/story/config.js（同上，返回预置的完整 DEMO 故事配置文档）
  ★ 此阶段不需要任何 AI 密钥，也不会产生任何 AI 费用

第三步：写游戏前端——模块三（约 1 天）
  ├─ 故事简介展示和确认界面（此时显示的是 Mock 数据，但界面完全真实）
  └─ 加载动画和错误提示

第四步：写游戏前端——模块四（约 3～5 天，工作量最大）
  ├─ 地图背景渲染（无资产时先用 Canvas 占位图）
  ├─ 玩家角色 + NPC 渲染与站位
  ├─ 点击移动（直线移动即可）
  ├─ 感叹号显示与交互触发
  └─ 对话气泡、三选一、错误回应、单词高亮
  ★ 全程使用 Mock 数据，可反复测试，零费用

第五步：写游戏前端——模块五（约 0.5 天）
  └─ 评分面板、星级动效、三个按钮
  ★ 至此游戏本体测试完毕，整体流程跑通

第六步：接入真实 AI（约 1 天）
  ├─ 注册 DeepSeek 账号，获取 API Key
  ├─ 在 Vercel 环境变量中填入 AI_API_KEY / AI_BASE_URL / AI_MODEL
  ├─ 将 USE_MOCK_AI 改为 false，重新部署
  └─ 测试真实 AI 生成效果，检查 JSON 格式校验逻辑是否正常工作

第七步：写管理页（约 1 天）
  └─ Prompt 模板的列表、新增、切换启用版本

第八步：完整流程测试 + 上线
  ├─ 测试完整游戏流程（开发指南第 16.3 节有验证清单）
  └─ 绑定自定义域名（可选）
```

## 八、关键注意事项

开发过程中最容易遗漏或出错的地方：

### 关于 AI 接入策略

- **先开发游戏本体，后接 AI。** 开发全程先把 `USE_MOCK_AI` 环境变量设为 `true`，接口返回预置的固定数据，游戏照常运行。等游戏本体测试完毕，再把这个开关改为 `false` 接入真实 AI，避免调试期间反复消耗 API 费用。
- **推荐使用 DeepSeek**，费用约为 OpenAI 的 1/10，且国内可直接访问无需代理。接口格式与 OpenAI 完全兼容，日后想换服务商只需在 Vercel 环境变量里改一下 `AI_BASE_URL` 和 `AI_MODEL`，代码不需要改动。
- **AI 密钥只存在 Vercel 环境变量里**，绝对不能写进代码文件，不然推送到 GitHub 后会泄露。
- **本地开发时** 在项目根目录建 `.env.local` 文件写入 `USE_MOCK_AI=true`，不需要 API 密钥也能完整运行游戏。这个文件**不要提交到 GitHub**（在 `.gitignore` 里加上它）。
- **AI 生成可能失败**，每个接口必须写重试逻辑（最多 3 次），3次都失败就返回预置 DEMO 配置，不能让页面卡住。
- **AI 返回的 JSON 必须经过校验**，不合法的数据不能直接进入游戏，否则游戏会崩溃（校验规则详见开发指南第 7.4 节）。
- **Prompt 不能硬编码**，必须从 Supabase 的 `prompt_templates` 表中动态读取当前启用的版本，这样你改 Prompt 不需要重新部署代码。

### 关于数据安全

- 在 Supabase 中为**所有业务表开启 Row Level Security（RLS）**，否则所有用户都能看到彼此的数据。
- **Supabase service_role key**（服务端密钥）只能存在 Vercel 环境变量里，不能出现在前端代码中。前端只能使用 `anon key`（公开密钥）。

### 关于游戏逻辑

- **对话必须严格线性**：玩家点击没有感叹号的 NPC 不应有任何响应。
- **错误次数必须在整个游戏过程中持续累计**，直到结算时一次性展示，不要每轮重置。
- **"再玩一次"** 是重新加载同一份配置文档，不是重新生成故事。
- **单词包中的词**在所有 NPC 台词、选项文本、翻译文本中都要保持红色加粗高亮。

### 关于资源（地图/角色图片）

- 正式美术资产没有之前，**用 Canvas 程序生成占位图**（彩色色块+文字）代替，不要用空图或者报错，游戏流程必须完整可运行。
- 图片规格：地图 750×1200px，NPC 100×160px，玩家 80×120px（详见开发指南第 12 节）。

## 九、管理员日常操作手册

上线后你需要定期维护的事情：

| 你想做什么 | 去哪里操作 |
|-----------|---------|
| **修改 AI 提示词（Prompt）** | 游戏网址 `/admin` → Prompt 管理页 → 新增版本 → 切换启用 |
| **更换 AI API 密钥** | vercel.com → 项目 → Settings → Environment Variables → 修改 `AI_API_KEY`（和 `AI_BASE_URL`、`AI_MODEL`）→ 重新部署 |
| **给某个用户升级为 VIP** | supabase.com → Studio → `users` 表 → 找到用户 → 把 `tier` 字段改为 `vip` |
| **重置某用户的游玩次数** | Supabase Studio → `users` 表 → 把 `quota_used` 字段改为 `0` |
| **禁用某个账号** | Supabase Studio → Authentication → Users → 找到用户 → Disable |
| **查看某条 AI 生成的故事** | Supabase Studio → `story_configs` 表 → 点击对应行 → 看 `config_json` 字段 |
| **查看游戏记录数据** | Supabase Studio → `game_sessions` 表 |
| **更新游戏代码** | 用 GitHub Desktop 修改后 Commit + Push → Vercel 自动部署（约1分钟） |
| **数据备份** | Supabase Studio → Database → Backups（免费套餐自动每日备份，保留7天） |

## 十一、费用预估

| 阶段 | 预计月费 | 说明 |
|------|---------|------|
| **DEMO 开发阶段** | $0 | Vercel 和 Supabase 免费套餐完全够用 |
| **小规模上线（日活 < 100 人）** | ~$1 以内 | 主要是 DeepSeek API 调用费用（费用约为 OpenAI 的 1/10） |
