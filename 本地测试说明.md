# 本地测试说明（前端 + 后端一起跑）

## 一、安装 Node.js（若尚未安装）

本机需要先安装 Node.js，任选一种方式：

- **方式 A**：打开 [https://nodejs.org](https://nodejs.org) 下载并安装 LTS 版本  
- **方式 B**：若已安装 Homebrew，在终端执行：`brew install node`

安装后在终端执行 `node -v`，能显示版本号即表示成功。

---

## 二、安装依赖并启动

在项目根目录打开终端，执行：

```bash
cd /Volumes/SC/work/AI/program/WordQuest
npm install
npm run local
```

第一次运行 `npm install` 会下载 Vercel CLI，可能需等待几十秒。  
`npm run local` 启动后，终端会显示本地地址，例如：

```
Ready! Available at http://localhost:3000
```

---

## 三、在浏览器里测试

1. 打开浏览器，访问终端里显示的地址（如 `http://localhost:3000`）。
2. **测游戏**：直接使用该地址，或点登录/注册、选词书、开始游戏等。
3. **测管理后台**：在地址后加 `#admin`，例如：  
   **http://localhost:3000/#admin**  
   - 输入管理员密码（未配置时默认为 `admin123`）  
   - 进入后可查看用户列表、在线人数、添加/删除用户、调整等级等。

---

## 四、用户数据持久化（Vercel Blob，可选）

若希望添加的用户在重启服务后仍保留，需配置 Vercel Blob：

1. 在 [Vercel Dashboard](https://vercel.com) 进入你的项目 → **Storage** → **Create Database** → 选择 **Blob**，创建 Store。
2. 本地拉取环境变量：在项目根目录执行 `vercel env pull .env.local`（会生成 `.env.local`，内含 `BLOB_READ_WRITE_TOKEN`）。
3. 重新执行 `npm run local` 启动。此时用户数据会写入 Blob，多请求共享、重启不丢失。

未配置时，数据仅存内存，重启或不同请求可能看不到刚添加的用户。

---

## 五、可选：自定义管理员密码

在项目根目录新建 `.env` 或 `.env.local` 文件，写入：

```
ADMIN_PASSWORD=你的密码
```

保存后重新执行 `npm run local`，再用新密码登录管理后台即可。

---

**简要步骤：** 安装 Node → 在项目根目录执行 `npm install` 和 `npm run local` → 浏览器打开终端显示的地址，测游戏或 `地址/#admin` 测管理后台。
